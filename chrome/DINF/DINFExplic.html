Menu Apps
This is the application menu. You can open applications: "Open App", save applications: "Save App" or save applications with a new name: "Save App As".
An application is a file describing a set of a Notation 3 (N3) data file, a file containing queries in Notation 3 format and parameters. The application file has the suffix .app and is also in format N3.
Notation 3 has a central role in this program. It was developed by Tim Berners-Lee
An explanation of his hand can be found on: Notation 3

The 'Exit' menuitem first asks the user if does not want to save the application. If he says 'Yes' then the application save process is executed and then the system exits. If he says 'No' the system exits immediately.

Menu Wizards
The 'Demo' menuitem uses alert boxes, might seem somewhat strange but you can quickly learn the most important things.
The 'QuickStartWizard' menuitem gives more or less the same information, but uses the XUL wizard system.

The 'Application Flow Chart' menuitem show a flow chart that demonstrates how to create an application.

Menu Web Services
The 'Start Server' menuitem starts the web service server.

The 'Stop Server' menuitem stops the web service server.

The 'Save Web Service' menuitem saves the source code in the zone "Source Code" as a web service.
See 'Menu Help', item 'Web Services' for an explanation of the mechanism.

The 'Query Web Service' menuitem permits to query a web service who was created with the DINF system.
(No other web services!!!).

The 'Set Port' menuitem permits to change the TCP/IP port on which the web service server is listening.
The default is: 10000.

The 'View WSDL' shows the WSDL definition of the webservice.There is one unique web service for
querying all N3 files.

Menu Source Code
Though this product is intended to work with files in Notation 3 format, nevertheless with the menuitem 'Open' you can read any file. There is a filter wich selects the N3 files, but you can change this with eg *.txt in the file selection zone. You can then eventually edit this file or merge with other files and then save it with 'Save' or, if you want to change the name, with: 'Save As'.
With 'Save Queries' you can save the queries. If a name is known for the queryfile then it will be used. Otherwise a name will be asked for.
With 'Save Output' the output zone will be saved to a file. The system will always ask for a filename.
With 'Save Messages' the message zone will be saved to a file. The system will always ask for a filename.

Menu Print
With this menu different zones of the screen can be printed.
With 'Print source' the source zone will be printed.
With 'Print output' the output zone will be printed.
With 'Print messages' the message zone will be printed.

Menu Clear
With this menu different zones of the screen can be cleared.
With 'Clear output' the output zone can be cleared.
With 'Clear source' the source zone can be cleared.
With 'Clear messages' the message zone can be cleared.
With 'Clear query' the query zone can be cleared.

Menu Reasoners
The DINF package supports two reasoners: a forward reasoner and a backward reasoner. By default the forward reasoner will be used. Here however you can select also the backward reasoner.
With 'Forward reasoner' you select the forward reasoner.
With 'Backward reasoner' you select the backward reasoner.

Menu Help
With 'Content' this text is displayed.
With 'About' the acronym DINF is explained (Distributed Inferencing) and the version of the system is given (1.0). Other contents of the screen
Now I will say something about the textareas, lists and buttons that are on the screen.

Left part of the screen
Under the heading 'Most recent apps' a list is maintained of the most recently opened apps.
The number of apps in that list is limited to 10.
Under the list is a button 'Load app' for laoding an application and a button
'Delete app' for deleting an application from the list.The application itself
will not be deleted!

Under these buttons there is the textarea labeled 'Output'. Here the output of queries is shown.
Under the textarea is a button 'Clear output' for clearing the textarea.


Central part of the screen
The textarea labeled 'Source code' is meant to contain N3(Notation 3 ) source code.
The user is naturally free to enter whatever he likes. The contents can be saved by using the menu 'Source Code'.
Under the 'Source code' textarea we have the 'Messages' textarea, where normally messages of the system will appear.
All textareas are writable and can be saved to disk. This can be handy for adding commentaries and keeping them.

Right part of the screen
The textarea labeled 'QUERIES' is used for querying the N3 source code that is in the 'Source code' area.
Queries are N3 statements that can contain variables; for more explanation see:
Inferencing with RDF and Notation 3 for beginners
The button 'Execute' under the textarea will execute the query and the results of the query will be exposed in the 'Output' textarea.
The button 'Add query' will add the query to the list of queries labeled 'Query list'.
This is usefull is there are queries you want to use regularly, also for querying web services.
The list of queries can be saved to disk.

The we have the query list. Queries can be saved in this list by giving them a name. A query can be sleceted by its name and
moved to the 'QUERIES' textarea for execution. A window with a list of available queries can be obtained.
There are quit some manipulations possible on the queries:
- the button 'Select query' moves the query that is selected in the list to the 'QUERIES' textarea.
- the button 'Delete query' removes the selected query from the list.
- the button 'Rename query' gives a different name to teh selected query.
- the button 'Show queries' will show the list of names and their associated query in a separate window.
- the button 'Save' will save the list of queries to disk.
- the button 'Restore' will fill the list by reading a query file from the disk.
